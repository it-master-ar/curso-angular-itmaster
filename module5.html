Modules, Scope & Controllers

Modules

  • Contenedores para componentes de coódigo que pertenezcan a una misma funcionalidad.
  • Pueden contener, controllers, directives, filters, services, providers, etc.
  • Pueden depender de otros modulos (Dependency Injection)
  • Modulo principal (ng-app)

    • Ventajas

    • Declaración fácil de entender.
    • Reutilización de código.
    • Carga en paralelo (no importa el orden)
    • Facilitación de testeo.

    Sintaxis

              
            // Setter
            angular.module('myApp', []);
            angular.module('myApp', ['ngRoute']);

            // Getter
            angular.module('myApp');

    Best Practices

  • Mantener los modulos tan chicos como sea posible.
  • Isolados y sin acomplamiento, maximizar la reutilización.
  • Usar encadenamiento, evitar la asignacion de variables.

    • Demo

    .run()

    • Método de inicialización (post startup).
    • Puede haber N llamadas a run().
    • Útil para codigo inicial (metodo main).

    $rootScope

    Scope padre (ng-app)

    Sistema jerarquico (arbol)

    Scope ($scope)

    • Objetos jerarquicos de almacenamiento tipo clave valor.
    • Similar a herencia CSS o JavaScript.
    • Componente principal del motor de binding.
    • "Pegamento" que permite asociar un valor/variable con una vista (HTML).

    Caracteristicas

    • $watch observar mutaciones
    • $apply propagar cambios (fuera del mundo Angular)
    • Puende ser isolados
    • Dan contexto a las expresiones

    Controllers

    • Función constructora que define y expande scopes.
    • ng-controller="MyController"
      • Un controller debe pertener a un module.

    Sintaxis

              
            angular.module('myApp', [])
                .controller('MyController', function($scope) {
                    $scope.version = 1.1;
                });
